CY.PROMPT() PHILOSOPHY:
cy.prompt() enables natural language test steps that adapt to page changes. The AI agent interprets instructions and finds the best way to accomplish each step, making tests more resilient to UI changes.

CYPRESS API AVAILABLE (Use with cy.prompt()):
- All standard Cypress commands: visit, get, contains, click, type, select, check, etc.
- Assertions: should(), expect(), assert()
- Network: intercept(), wait(), request()
- Navigation: go(), reload(), url(), location()
- Waits: wait(), should('be.visible'), should('exist')
- Storage: getCookie(), setCookie(), clearLocalStorage()
- Window: window(), document(), title()
- Files: fixture(), readFile(), writeFile()
- Advanced: within(), then(), invoke(), its(), each(), wrap()

GUIDELINES:
1. Use natural language prompts that describe WHAT to do, not HOW
2. Break complex workflows into clear, atomic steps
3. Use fixture data for dynamic values (NOT hardcoded!)
4. Include verification steps between actions
5. Add assertions to validate outcomes

7. Combine cy.prompt() with standard Cypress for power + flexibility

PATTERN STRUCTURE:

```javascript
describe('Tests', function () {{
    beforeEach(function () {{
        cy.fixture('url_test_data').then((data) => {{
            this.testData = data;
        }});
    }});

    it('should succeed with valid data', function () {{
        // Navigate
        cy.visit(this.testData.url);
        
        // Get test data
        const valid = this.testData.test_cases.find(tc => tc.name === 'valid_test');
        const selectors = this.testData.selectors;
        
        // Use cy.prompt() for flexible element interaction
        cy.prompt([
            "Wait for the page to fully load",
            "Find the email input field and type the email address",
            "Find the password input field and type the password",
            "Click the submit button",
            "Verify successful login by checking for dashboard or success message"
        ]);
        
        // OR use traditional Cypress with dynamic selectors
        Object.keys(selectors).forEach(field => {{
            if (field !== 'submit' && valid[field]) {{
                cy.get(selectors[field]).should('be.visible').type(valid[field]);
            }}
        }});
        
        cy.get(selectors.submit).click();
        
        // Assertions
        cy.url().should('include', valid.expected || 'success');
    }});

    it('should fail with invalid data', function () {{
        cy.visit(this.testData.url);
        const invalid = this.testData.test_cases.find(tc => tc.name === 'invalid_test');
        const selectors = this.testData.selectors;
        
        // Dynamic form filling
        Object.keys(selectors).forEach(field => {{
            if (field !== 'submit' && invalid[field]) {{
                cy.get(selectors[field]).type(invalid[field]);
            }}
        }});
        
        cy.get(selectors.submit).click();
        
        // Verify error state
        cy.prompt(["Verify that an error message is displayed"]);
        // OR
        cy.url().should('include', invalid.expected || 'error');
    }});
}});
```

COMBINING CY.PROMPT() WITH STANDARD CYPRESS:

```javascript
it('hybrid approach', function () {{
    cy.visit(this.testData.url);
    const valid = this.testData.test_cases.find(tc => tc.name === 'valid_test');
    
    // Use cy.prompt() for flexible navigation
    cy.prompt(["Navigate to login section"]);
    
    // Use standard Cypress for precise control
    cy.get(this.testData.selectors.username).type(valid.username);
    cy.get(this.testData.selectors.password).type(valid.password);
    
    // Use cy.prompt() for submission and verification
    cy.prompt([
        "Click the login button",
        "Wait for successful login",
        "Verify user is on dashboard"
    ]);
    
    // Standard assertion
    cy.url().should('include', '/dashboard');
}});
```
